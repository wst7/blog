情景

在使用git命令操作仓库代码时，建议大家不要心急，敲完命令后先别急着敲回车(虽然命令+回车连着敲起来很爽)，检查一下命令是否正确，名称（分支）是否是正确的单词，待检查完没有问题，便可以狠狠的砸下Enter键。

如果检查不够仔细，创建的分支名称不想要，当然也有对应的修改方法，下面将针对两种情况分别作出处理：

  • 分支只在本地，没有推送到远程
  • 分支已经推送到远程
